# Vocabulary source data

## ⚠ `master.xlsx` is gitignored — keep your own backup

This folder's `master.xlsx` is the **source of truth** for all vocabulary but is
**never committed** (SPEC §11). GitHub is not backing it up — keep a copy in
your own cloud drive or backup system.

Because the workbook can't reach CI, the compiled outputs
(`src/content/vocabulary/words.json`, `groups.json`, `audio-credits.md`) **are
committed** — after editing the workbook, run `npm run data:all` and commit the
regenerated files.

## Workbook contract

One sheet per topic group (27), header row, columns in order:

| Column | Rules |
|---|---|
| `ID` | Stable, group-prefixed (`travel_excursión`), assigned once and frozen. Doubles as the audio filename stem (`<id>_1/2/3.mp3`) and the saved-progress key — renaming an ID orphans that word's audio *and* its SRS progress (SPEC §4). |
| `Spanish (Spain)` | Must match the manifest's `Spanish` column (converter enforces). |
| `English (US)` | Exact-match duplicates across the workbook auto-populate `relatedWords`. |
| `Attributes` | One of `[sust.] [verb.] [adj.] [adv.] [pron.] [prep.] [conj.] [interj.] [sust./adj.]` — anything else fails the build. |
| `Gender` | One of `[] [masc.] [fem.] [neut.] [invar.] [masc./fem.]`. |
| `Phonetics` | Slash-delimited IPA (`/ekskuˈɾsjon/`) or empty. |
| `Note` | Optional register tag, e.g. `(informal)`. |
| `CEFR Level` | `A1` `A2` `B1` `B2`. |
| `Example` | Sentence with the target word **bolded** (rich-text runs → `**word**` in JSON). |

**Scripts that EDIT the workbook** must use
`openpyxl.load_workbook(..., rich_text=True)` or all 2001 bold runs in the
Example column are silently flattened on save (SPEC §14). Read-only access
(like our converter) is safe.

## Files here

- `audio-manifest.csv` — per-word audio outcomes from the audio thread (which
  files exist, WordReference notes, Lingua Libre speaker per recording). Input
  to the converter and validators; don't regenerate.
- `phonetics-report.csv` — which IPA entries are exact vs. composed (informational).
- `related-manual.json` — hand-maintained related-word links that share no
  literal English text (e.g. saber/conocer): `{ "word_id": ["other_id", ...] }`.
  Applied symmetrically.
- `disambiguation-notes.json` — owner-approved trainer hints keyed by word id.
  Drafted via the SPEC §4 pipeline (Round 6), reviewed before shipping.
- `disambiguation-report.md` — **generated** checklist of auto-linked groups
  still missing notes. Regenerated by `npm run data:convert`.

## Audio licensing (SPEC §14)
