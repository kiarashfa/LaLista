## Development

When starting the dev server, use background mode:

```
astro dev --background
```

Manage the background server with `astro dev stop`, `astro dev status`, and `astro dev logs`.

## LaLista project notes

- **Contract docs**: SPEC.md (product decisions — nothing there is a default to
  second-guess) and Grammar.md (grammar content format). Read
  before changing behavior. Owner-confirmed additions: site name is **LaLista**
  (not ListO), Test timers 7s/question + 90s pool, workbook thresholds 80/50.
- **Data flow**: `data/` is human-edited source of truth; `src/content/vocabulary/`
  is GENERATED by `npm run data:convert` but COMMITTED (master.xlsx is gitignored,
  CI can't regenerate). Grammar collections load directly from `data/grammar/`
  via glob loader in `src/content.config.ts`.
- **Never commit**: `data/vocabulary/master.xlsx`, `public/audio/*_1.mp3`
  (WordReference, personal-use only per SPEC §14) — both gitignored; verify with
  `git check-ignore` after touching .gitignore.
- **After editing the workbook**: `npm run data:all` (convert + credits + validate).
- Design source of truth: `docs/mockups/tokens.css` (+ two mockup HTML pages).
